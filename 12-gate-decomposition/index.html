Gate Decomposition

Breaking Down Complex Gates into Simple Operations

Prerequisites

1. Rotation Gates

Rx(θ), Ry(θ), Rz(θ) rotate the state around X, Y, Z axes by angle θ.

2. Euler Angles

Any 3D rotation can be described by three angles (α, β, γ) - rotations around different axes.

3. Matrix Multiplication

Decomposition means finding gates G₁, G₂, ... such that U = G₁ · G₂ · ...

ZYZ Decomposition

Any single-qubit unitary U can be written as:

U = e^(iα) · Rz(β) · Ry(γ) · Rz(δ)

Global phase e^(iα) is often ignored (unobservable)

Euler Angle Explorer

|ψ⟩ = |0⟩

Decomposition Steps

1
Start with |0⟩ - The north pole of the Bloch sphere
2
Apply Rz(δ) - Rotate around Z axis (changes phase, |0⟩ stays at north pole)
3
Apply Ry(γ) - Rotate around Y axis (moves state toward equator)
4
Apply Rz(β) - Final Z rotation (adjusts longitude on sphere)

Common Gate Decompositions

Hadamard Gate

H = Rz(π) · Ry(π/2)

Or equivalently: H = (X + Z)/√2

X Gate (Pauli-X)

X = Ry(π)

Or: X = iRx(π) (up to global phase)

S Gate

S = Rz(π/2)

S is just a π/2 rotation around Z

T Gate

T = Rz(π/4)

T is a π/4 rotation around Z

Two-Qubit Gate Decomposition

CNOT Decomposition

CNOT can be built from single-qubit gates and a different two-qubit gate (CZ):

CNOT = H₂ · CZ · H₂

H₂ means Hadamard on qubit 2 (target)

When to Decompose Gates

Hardware only supports Rz, Ry: Use ZYZ decomposition for any single-qubit gate
Need to implement custom rotation: Calculate Euler angles (β, γ, δ) for your target unitary
Optimize circuit depth: Combine adjacent rotations: Rz(a)·Rz(b) = Rz(a+b)
Hardware has native CZ but not CNOT: Use CNOT = H·CZ·H decomposition
